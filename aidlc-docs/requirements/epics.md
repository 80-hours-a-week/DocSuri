# DocSuri MVP — Epic 정의

- **결정 근거**: [`aidlc-docs/plans/user_stories_plan.md`](../plans/user_stories_plan.md) 부록 A · 결정 3.A
- **MVP 범위**: [E1](#e1) · [E2](#e2) · [E3](#e3) + [E4](#e4) 일부 (인용 1-hop 시각화만)
- **MVP 비포함(Won't this cycle)**: [E5](#e5) 재현성 자동 평가, [E6](#e6) 알림/트렌드 데일리, [E7](#e7) 개인화·필터 저장
- **디바이스 정책**: 모든 Epic은 **모바일·태블릿·데스크톱 3개 폼팩터를 1차 지원**한다. Epic별 반응형 분기·터치 친화·모바일 응답 목표는 [`requirements/nfr.md`](nfr.md)의 `NFR-MOBILE-*` / `NFR-NET-*` / `NFR-A11Y-03` 키로 위임하며 각 사용자 스토리에서 인용한다.

<a id="e1"></a>
## E1. Discover — 검색·발견

| 항목 | 값 |
|---|---|
| 목적 | 사용자가 자연어로 표현한 연구 의도를 의미 유사도 기반으로 논문 코퍼스에 매칭한다. |
| 핵심 능력 | 자연어 쿼리, 키워드 자동 확장, 정렬·필터, 결과 메타데이터 표시 |
| 관련 페르소나 | [P1](../story-artifacts/personas.md#p1) 박지훈(전문), [P2](../story-artifacts/personas.md#p2) 김민서(입문) |
| 데이터 의존 | 공개 논문 코퍼스(arXiv 등) + 임베딩 인덱스 |
| 성공 지표 | P50 응답 < 3초 · 상위 5건 내 관련 논문 ≥ 3건(체감) |

### MVP 범위
- 자연어 쿼리 → 임베딩 유사도 기반 상위 N건 반환.
- 정렬: 유사도/인용수/최신성. 필터: 발표 연도, 분야 태그.
- 학부 페르소나 대응: **난이도 추정 점수** 표시(가중치만, 본격 분류는 Won't).

### 범위 밖
- 다국어 동시 검색의 고급 처리. 영어/한국어 자연어 입력만 1차 지원.

<a id="e2"></a>
## E2. Comprehend — 이해·해독

| 항목 | 값 |
|---|---|
| 목적 | 발견한 논문을 사용자가 빠르게 이해할 수 있도록 변환한다. |
| 핵심 능력 | 요약, 번역, 독자 수준 적응 설명, 시각자료 설명 |
| 관련 페르소나 | [P1](../story-artifacts/personas.md#p1)(전문 요약), [P2](../story-artifacts/personas.md#p2)(쉬운 설명·번역) |
| 데이터 의존 | 논문 PDF/URL → 텍스트 추출 → LLM 요약·번역 파이프라인 |
| 성공 지표 | 페르소나별 가독성 점수 만족 · 요약 P95 < 20초 |

### MVP 범위
- 논문 PDF 또는 URL 입력 → 한국어 핵심 요약 (전문 모드 / 학부 모드 분기).
- 영문 본문 한국어 번역 (부분 발췌 단위).
- 학부 모드: 어려운 수식·약어를 학부 수준 한국어로 풀어쓰기.

### 범위 밖
- 표·그림 자동 캡션 생성은 *Could*. PDF 그림 영역 검출까지는 가지 않는다.

<a id="e3"></a>
## E3. Differentiate — 차별화·연구 공백

| 항목 | 값 |
|---|---|
| 목적 | 사용자의 연구 주제 초안과 기존 논문 간의 유사·차이·공백을 드러낸다. |
| 핵심 능력 | 유사도 분석, 중복 아이디어 검출, 연구 공백(gap) 후보 제시 |
| 관련 페르소나 | [P1](../story-artifacts/personas.md#p1)(졸업논문 novelty 검증), [P2](../story-artifacts/personas.md#p2)(가벼운 학부 수준 검토) |
| 데이터 의존 | 검색 결과 상위 N건 + 사용자 입력 초안의 임베딩 비교 |
| 성공 지표 | P1: novelty 보고서 1쪽 분량 산출 · P2: "비슷한 학부 프로젝트가 있는가" 1문단 응답 |

### MVP 범위
- 사용자 연구 주제 초안(텍스트) 입력 → 상위 유사 5건 + **차별성 요약 노트**.
- 연구 공백 후보 *제안 3개*까지 (LLM 보조).

### 범위 밖
- 자동 인용 추천(citation recommendation)은 [E4](#e4)·후속 사이클로.

<a id="e4"></a>
## E4. Trace — 인용 흐름 (MVP는 일부)

| 항목 | 값 |
|---|---|
| 목적 | 한 논문을 중심으로 기술 흐름을 시각적으로 보여준다. |
| 핵심 능력 | 인용 1-hop 시각화 + 영향력 큰 후속 논문 Top-K |
| 관련 페르소나 | [P1](../story-artifacts/personas.md#p1)(선행 연구 식별), [P2](../story-artifacts/personas.md#p2)(영향력 큰 후속 학습) |
| 데이터 의존 | 공개 인용 그래프 (Semantic Scholar API 등) |
| 성공 지표 | 1-hop 그래프 P95 < 5초 |

### MVP 범위
- 선택한 논문의 **직접 인용**(outgoing) + **피인용**(incoming) 1-hop만.
- 학부 모드: 그래프 대신 "후속 영향 Top 3" 리스트 단순화.

### 범위 밖
- 다단계 인용 트리, 시간 축 애니메이션, 분야 군집 컬러링은 후속 사이클.

## MVP 제외 Epic (Won't this cycle)

| Epic | 사유 |
|---|---|
| <a id="e5"></a>E5 Trust (재현 가능성 자동 평가) | 코드·데이터 링크 자동 수집·검증의 비용이 데모 범위를 초과. |
| <a id="e6"></a>E6 Stay-Current (알림/트렌드 데일리) | 백엔드 스케줄러·구독 모델·푸시 채널 필요. 인프라 비용 큼. |
| <a id="e7"></a>E7 Personalize (영속 개인화: 필터 프리셋 저장·관심 분야 학습·재방문 컨텍스트 복원) | **재방문·장기 컨텍스트가 필요한 영속 개인화**는 [NFR-SEC-01](nfr.md#nfr-sec-01)(비로그인 + PDF 임시 저장)과 정렬되지 않음. [Prompt 4](../prompts.md#prompt-4)·[5](../prompts.md#prompt-5)(2026-06-10) 검토 후 MVP 비포함으로 동결. |

> 비포함 Epic의 가치는 인정하되 본 사이클에서는 의도적으로 미루며, 후속 사이클의 우선 후보로 명시한다.

### 세션 한정 개인화는 어디에?

[E7](#e7)을 비포함으로 두더라도, **세션 한정 개인화**는 다음 스토리에 이미 흡수되어 있다 ([Prompt 4](../prompts.md#prompt-4) 검토 결과):

| 형태 | 위치 | 저장 매체 |
|---|---|---|
| 페르소나 모드 토글 (전문/학부) | [US-COMP-01](../story-artifacts/user_stories.md#us-comp-01) / [US-COMP-03](../story-artifacts/user_stories.md#us-comp-03) | 세션 메모리 |
| 사용자 주제·프로젝트 컨텍스트 주입 | [US-DIFF-01](../story-artifacts/user_stories.md#us-diff-01) / [US-DIFF-03](../story-artifacts/user_stories.md#us-diff-03) | 세션 메모리 |
| 필터·정렬 상태 직렬화 | [US-DISC-02](../story-artifacts/user_stories.md#us-disc-02) | URL 쿼리 |

설계 단계에서는 위 세 가지를 **세션 스코프 상태 모델**로 일관되게 다루고, 영속 개인화로의 확장 경로(localStorage → 익명 세션 ID → 정식 인증)는 후속 사이클의 옵션으로 열어둔다.
