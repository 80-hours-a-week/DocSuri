# DocSuri — 페르소나

**단계**: INCEPTION → 사용자 스토리(User Stories) · **일자**: 2026-06-15
페르소나 3종: **P1**(주), **P2**(부), **OP**(운영/유지보수 — 신뢰성/비용/AI 인시던트 담당).

---

## P1 — 현역 AI/ML 연구자 / 박사과정 *(주, PRIMARY)*
**이름**: 박지훈 (28) · AI 대학원 박사과정

**맥락**: 졸업논문을 써야 하는데 이미 엄청나게 많은 논문들과 하루에도 수백 건 이상의 논문들이 쏟아지는 상황에서 자신이 쓰려는 주제가 이미 있는 건지 알 수 없고 방대한 양으로 다 읽어 트랜드를 파악할 수 없다. 주 10시간 이상 연구 문헌을 조사하지만 겨우 찾은 논문조차 재현이 불가능해 실제 실험이나 논문 작성에 투자할 시간이 부족하다.

- **역할**: 특정 AI/ML 세부 분야를 추적하는 박사과정 연구자; 매일 논문을 읽고 선별하며, 종종 다른 일 사이사이 폰으로 본다.
- **목표**: 한 주제에 대해 가장 관련성 높은 + 최신 논문을 빠르게 발견; 아이디어가 이미 다뤄졌는지 확인(신규성 점검); 신뢰할 수 있는 읽기 목록 유지.
- **불편(Frustrations)**: 물량 과부하(하루 수백 건의 신규 논문); 주제 신규성에 대한 불확실성; 저효율 검색에 낭비되는 시간; 신뢰 불가/재현 불가 결과.
- **성공**: 몰랐던 관련·최신 논문을 폰에서 수초 내에, 신뢰할 수 있는 인용(날조 없음)과 함께 발견.

## P2 — 초기 문헌 리뷰 중인 대학원생 *(부, SECONDARY)*
- **역할**: AI/ML 문헌 리뷰를 시작하는 석사/박사 초기 학생.
- **맥락**: P1보다 도메인 숙련도가 낮음; 동일한 디스커버리 흐름을 더 가벼운 프레이밍으로 필요; 세부 분야의 기초 이해를 쌓는 중.
- **목표**: 새 주제의 핵심 논문 지도화; 시작용 라이브러리 구축.
- **불편**: 주요 논문이나 적절한 키워드를 모름; 쉽게 압도됨.
- **성공**: 자연어 주제를 입력해 읽고 저장할 신뢰할 만한 시작 논문 세트를 받음.

## OP — 운영자 / 유지보수자 *(최종 사용자 아님)*
- **역할**: DocSuri를 프로덕션에서 신뢰성·예산 내·신뢰 가능하게 유지하는 팀원.
- **맥락**: 공개·중간 티어·비용 상한 서비스의 복원력·비용·AI 인시던트 태세(RES-*, NFR-C1, NFR-R*, NFR-O1)를 담당.
- **목표**: 서비스를 가용 상태·월 비용 상한 내로 유지; 세 가지 AI 인시던트 클래스 — **비용 폭발**, **할루시네이션**, **반쪽짜리 결과(partial/half-baked)** — 를 사용자 피해 이전에 탐지·대응.
- **불편**: 조용한 오답; 폭주하는 LLM 지출; 우아한 폴백 없는 업스트림(arXiv/LLM/인덱스) 장애.
- **성공**: 인시던트가 신호+경보로 탐지되고, 시스템이 조용히 실패하는 대신 우아하게 저하되며, 지출이 상한을 절대 넘지 않음.
