# 연구 에이전트 사용자 스토리 실행 평가

**단계**: INCEPTION → 사용자 스토리(Part 1: 평가)
**일자**: 2026-06-24
**대상**: 연구 에이전트 (대화형 문헌탐색·근거형성 / 아이디어 novelty) — 유닛 번호 미배정

## 요청 분석
- **원 요청**: 네비바 검색↔마이페이지 사이의 대화형 연구 보조 — 여러 논문을 교차 확인해 근거를 정리(모드 A)하고, 사용자가 떠올린 주제가 이미 있는지 유사논문 비교(모드 B)로 알려준다. 문서 첨부 가능.
- **요구사항 입력**: `requirements.md`의 FR-22, FR-23, FR-24, FR-25, NFR-P5, NFR-C1 Agent 보강, QT-8, §12 Agent 카브아웃, C-2 경계.
- **사용자 영향**: 직접. 신규 대화형 진입 표면(하단 네비 전용 메뉴), 다논문 근거 정리 결과, 세션 재열람, 문서 첨부 흐름.
- **복잡도**: 표준 이상. 다논문 검색·교차확인, 추출 기반 근거 정리, 근거화/기권, 결과·세션 영속, 비차단 저하, 비용 게이트가 함께 필요하다.
- **이해관계자**: P1 연구자(주), P2 대학원생(부), OP 운영자(비용·근거화·AI 인시던트).

## 충족된 평가 기준
- [x] **High Priority — 신규 사용자 기능**: 대화형 근거형성은 사용자 대면 신규 기능이다.
- [x] **High Priority — 사용자 워크플로우 변경**: 네비에 전용 진입점이 추가되고 새 대화·첨부·세션 흐름이 생긴다.
- [x] **High Priority — 복잡한 비즈니스 로직**: 다논문 교차확인·근거화·기권·저하·비용 게이트가 다중 시나리오를 가진다.
- [x] **Medium Priority — 보안/프라이버시 영향**: 첨부 업로드 검증, owner-scoped 세션·결과, 보관/삭제 제어가 필요하다.
- [x] **Medium Priority — 복원력 영향**: 외부 의존/LLM 실패 시 부분 결과·저하, 본 검색 비차단이 필요하다.
- [x] **품질 기준 필요**: 근거 출처 정확도·날조 0·저하 처리를 QT-8 기준으로 검증해야 한다.

## 결정
**사용자 스토리 실행**: 예

**근거**: 연구 에이전트는 §12 frozen 범위를 여는 신규 사용자 대면 기능으로, 추출·근거화 경계(C-2)·기권·비차단 저하가 제품 신뢰에 직결된다. 스토리와 인수 기준으로 명시해야 v1(모드 A) 구현 범위와 모드 B(novelty) 차기 경계가 흔들리지 않는다.

## 기대 산출
- 연구 에이전트 전용 에픽(에픽 9)과 얇은 사용자 스토리.
- 기존 P1/P2/OP 페르소나 확장 매핑(새 페르소나 없이).
- FR-22~25, NFR-P5, NFR-C1 Agent 보강, QT-8, §12 카브아웃·C-2 경계 추적성.
- **모드 A는 v1 구현 스토리, 모드 B(novelty)는 차기 사이클 경계**가 인수 기준에서 명확히 드러남.
- 생성 산문·재현성 판정 제외, HOW(코퍼스 확장·외부 API·근거표 컬럼·모델·멀티턴·UI)는 Construction 이월이 스토리 altitude로 유지됨.
