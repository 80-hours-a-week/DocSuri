## 확장 규칙 준수 요약

- **Security Baseline**: 적용 대상. 스토리는 인증/소유자 범위 전제를 유지하고, 타임라인에 민감한 내부 로그가 노출되지 않게 하며, 파일 첨부를 검증해야 한다.
- **Resiliency Baseline**: 적용 대상. 스토리는 저하/실패 상태와 재시도/새 채팅 복구 경로를 포함해야 한다.
- **Property-Based Testing**: 적용 대상. Q16=B에 따라 속성이 있는 프론트엔드 로직에는 Full PBT를 적용하고, UI 렌더링은 예시 기반 테스트로 보완한다.
