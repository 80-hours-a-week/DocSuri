## Q1. 범위 편입 & 경계 (vs C-2 / §12) — 게이트 본체

요약/번역을 v1.x 범위에 **정식 기능으로 편입**하는가? 핵심은 C-2(생성 글쓰기 금지)와의 경계다 — "이미 검색돼 사용자가 탭한 *그 논문 1편*의 내용을 요약/번역"은 추출(FR-5 영역)이지, 사용자 본인 원고를 만드는 생성 글쓰기(C-2)가 아니다.

- **A) 편입한다 — 추출 경계 명시** (초안 권장): "검색된 그 논문의 내용 요약/번역"으로 한정. 사용자 원고/문헌리뷰 산문 생성은 계속 §12 제외 유지. §12에 "단, 검색된 단일 논문의 추출 요약/번역은 FR-12로 편입" 카브아웃 추가.
- **B) 편입하되 별도 사이클(v2)로 로드맵에만 기록** — 지금은 요구사항만 적고 Construction은 보류.
- **C) 편입하지 않는다** — 설계 초안을 로드맵 후보로만 보관.
- **X) 기타**

[Answer]: A — 편입하되 추출 경계 명시. 검색된 단일 논문의 추출 요약/번역만 FR-12로 편입, 사용자 원고 생성(C-2)은 §12 제외 유지.

## Q3. 성능(NFR) 프레이밍 — 검색 SLA 포함 여부

요약/번역은 결과 카드에서의 **온디맨드 액션**이라, 초안은 검색 지연 목표 **NFR-P1(P50<3s)** 대상이 아니라고 본다(체감은 스트리밍+캐시로 관리; 첫 생성은 수십 초 가능).

- **A) NFR-P1 대상 제외 — 별도 온디맨드 NFR 신설** (초안 권장): "캐시 히트=즉시 / 첫 생성=스트리밍 TTFB 목표" 식 별도 지표.
- **B) NFR-P1과 동일 SLA 적용** (검색과 같은 지연 목표를 요약에도 요구).
- **X) 기타**

[Answer]: A — NFR-P1 대상 제외, 별도 온디맨드 NFR 신설(캐시 히트=즉시 / 첫 생성=스트리밍 TTFB).

## Q4. 비용 상한(NFR-C1) 처리 — 신규 Sonnet 비용 라인

요약은 Sonnet 호출(건당 ≈ $0.1~0.2)이라 **현 비용표에 없는 신규 라인**이다(현재 NFR-C1 $1,600/월 상한, 사용 ~$370-400). 초안은 온디맨드+영구저장으로 "열어본 distinct 논문 × 1회"만 과금되어 bounded라고 본다.

- **A) 기존 $1,600 상한 안에서 흡수 + U6 비용게이트 재사용** (초안 권장): 예산 게이트(CostGuard) 초과 시 요약 일시 기권(FR-11 명시 상태). 비용 텔레메트리에 U7 라인 추가.
- **B) NFR-C1 상한을 상향** (금액은 별도 확정).
- **C) U7 전용 서브예산**을 별도로 둠.
- **X) 기타**

[Answer]: A — 기존 $1,600 상한 흡수 + U6 CostGuard 재사용. 예산 초과 시 요약 일시 기권(FR-11), 텔레메트리에 U7 라인 추가.

## Q5. 근거화/기권을 강제 인수기준으로

초안은 요약/번역 출력도 **엄격 근거화(FR-5/QT-1)** 적용 — 원문에 없는 주장은 날조로 보고 **기권**, 항목별 "출처 보기"(원문 앵커) 제공, U6 GroundingEnforcementHook 재사용을 제안한다.

- **A) 강제한다 — FR-12 인수기준에 명시** (초안 권장): 날조 0건·근거 없으면 기권·앵커 제공. QT-1 평가셋에 요약/번역 케이스 추가.
- **B) 완화** — 근거화는 권고 수준(기권 대신 경고 표기).
- **X) 기타**

[Answer]: A — 강제. 날조 0건·근거 없으면 기권·앵커 제공, QT-1 평가셋에 요약/번역 케이스 추가.

## Q6. 개인화 범위 경계 (요구사항 수준)

초안은 개인화를 비용으로 가른다: persona(전문가용/입문자용, 논문당 2벌 생성)·뷰 프리셋(추가비용 0)·용어집(P1 시드 + P2 개인 선호)은 **포함**; 커뮤니티 공유 용어집(P3, 모더레이션 필요)·자유입력 "내 연구노트 관련성"(진짜 per-user)은 **제외**.

- **A) 초안대로 — persona+뷰+용어집(P1/P2) 포함 / P3·자유입력 제외** (초안 권장).
- **B) 최소 — 개인화 없이 단일 공유 요약/번역만** v1, 개인화는 후속.
- **X) 기타**

[Answer]: A — persona(전문/입문) + 뷰 프리셋 + 용어집(P1/P2) 포함, P3(커뮤니티)·자유입력 per-user 제외.
