## Q1. 요약/번역 입력 형태 — 평문 → doc-model (게이트 D2)

요약(FR-12)·번역(FR-13)의 LLM 입력을 정규화 평문(`.txt`)에서 **구조화 doc-model**(섹션/블록·표=데이터·수식=LaTeX·그림 참조)로 전환하는가? (생성·근거화·캐시 **로직은 불변**, 입력만 업그레이드.)

- **A) doc-model로 전환 — 로직 불변** (게이트 D2): 표·수식을 무시하던 품질 갭 해소. 인덱스/검색 경로(chunk·embed)는 평문 유지(불변).
- **B) 평문 유지** — 현행.
- **X) 기타**

[Answer]: A — 요약/번역 입력 = doc-model. 생성/근거화/캐시 로직 불변. FR-12 개정(입력 형태), `.txt`는 인덱스·검색 투영으로 유지(Q4 게이트).

## Q2. 자체 리치뷰 등재 형식 — **실질 갈림길** (게이트 D4)

doc-model을 앱 안에서 렌더하는 **자체 리치뷰**(목차·KaTeX 수식·표 컴포넌트·그림 webp·앵커 점프)를 요구사항에 어떻게 등재하는가? D4로 "표시 전용 부산물"에서 "목적지 표면(요약/번역/주석/에이전트 통합·로그수집·개인화)"으로 격상된다.

- **A) 신규 FR-18로 1급 등재** (확정): 고유 수용기준(doc-model 렌더·TOC·수식·표 컴포넌트·앵커·OA 게이트)·추적성 분리. FR-17은 그림 이미지 표시로 좁힘.
- **B) FR-17/FR-12 보강으로 흡수** — 신규 FR 없이.
- **X) 기타**

[Answer]: A — 신규 **FR-18 자체 리치뷰**. PDF.js 픽셀 재현 아님(콘텐츠 재렌더, arXiv HTML 방식).

## Q3. 표 표현 — 크롭 이미지 → 구조화 데이터 (게이트 D8)

표(FR-17에서 "도표 이미지")를 크롭 이미지로 둘 것인가, **구조화 데이터(rows/cols)**로 둘 것인가?

- **A) 구조화 데이터** (게이트 D8): 표 숫자가 요약 LLM·근거화·에이전트에 가시(크롭 이미지는 텍스트 에이전트에 깜깜이). 그림은 이미지(webp) 유지. 비-HTML 폴백 시에만 표 크롭 이미지.
- **B) 크롭 이미지 유지** — 현행 FR-17.
- **X) 기타**

[Answer]: A — 표=데이터(doc-model). FR-17 개정(그림=이미지·표=데이터). 표 크롭은 PDF 폴백(~9%)만.

## Q4. PDF 원문 저장·다운로드 (게이트 D3)

PDF 원문을 S3에 저장하거나 다운로드 버튼을 제공하는가?

- **A) 미저장·다운로드 없음** (게이트 D3): 자산/doc-model 추출용 transient fetch만. arXiv 재취득 가능, PDF는 에이전트에 나쁜 포맷. 저장 이유 없음.
- **B) PDF 저장 + 다운로드 제공**.
- **X) 기타**

[Answer]: A — PDF 미저장. §12에 명시(다운로드 없음·transient fetch만).

## Q5. 비전(이미지) 추론 카브아웃 (게이트 D5)

그림 이미지를 비전 LLM으로 읽어 요약·근거에 반영하는 작업을 본 사이클에 편입하는가?

- **A) 제외 유지 — 에이전트 단계 on-demand 툴로 후속** (게이트 D5): webp 유지로 재수집 없이 후속 도입. batch 사전처리 안 함. §12 멀티모달 카브아웃(2026-06-22) 유지.
- **B) 본 사이클 편입**.
- **X) 기타**

[Answer]: A — 비전 추론 제외 유지. 그림 webp는 on-demand 비전 입력으로 보존(FR-17/§12 불변).

## Q6. 앵커 계약 — doc-model 결정적 id (Q3 스키마)

근거 앵커("출처 보기")가 가리키는 핸들을 무엇으로 하는가? (FR-12/QT-5 근거화·FR-18 리치뷰·에이전트 공통.)

- **A) doc-model Section/Block 결정적 id** (Q3 스키마 결정): `Anchor.target`=id(+선택 span). GroundingValidator 앵커 실재성 체크가 결정적·견고. 기존 "섹션 라벨+정규식" 취약성 해소.
- **B) 섹션 라벨+문자 span 유지**.
- **X) 기타**

[Answer]: A — 결정적 블록 id. FR-12 앵커·QT-5가 doc-model id로 검증. `summarization.schema.json` Anchor.target 의미 명확화(가산적).

## Q7. 생성 트리거·비용 (게이트 D6)

doc-model을 언제 생성하는가? (비용 영향.)

- **A) lazy on-demand + `(paperId, version)` 캐시** (게이트 D6): 첫 요약/열람/에이전트 사용 시 생성. 전 코퍼스 eager 아님 — 비용=사용량 비례. 인기 논문 일부 eager는 후속.
- **B) 인제스천 시 전 코퍼스 eager 생성**.
- **X) 기타**

[Answer]: A — lazy on-demand + 캐시. NFR-C1 비용은 사용량 비례(eager 백필 비용 회피). NFR-P2 온디맨드 정신과 일관.
