# 문헌탐색·근거형성 Agent — 요구사항 명확화 질문 (Requirement Verification — Literature & Evidence Agent)

**단계**: INCEPTION → Requirements Analysis (재인셉션 / 차터 **페이즈 4** = 문헌탐색·근거형성 Agent) · **일자**: 2026-06-28
**담당**: 화랑님
**대상 기능**: 로그인 필수 온디맨드 **대화형 다논문 문헌탐색·근거형성** — 사용자 질의/첨부에 대해 여러 논문을 검색·교차확인하여 **추출 기반 근거**(핵심 주장·방법·결과 수치·한계)를 **비교 정리**해 출처와 함께 제시하고, 근거가 없으면 **기권**한다. 결과·대화 세션을 owner-scoped로 영속(전용 네비 진입·세션 리스트 재열람). UI = **채팅 형식**(모드 선택).
**범위 밖(별개 유닛)**: 연구아이디어 novelty 비교·아이디어 생성 = **페이즈 5 연구아이디어 유닛**(석현님). 본 유닛은 5가 **Tool로 의존(D2 확정 A)** 하는 하부 근거형성 엔진이다.
**영향 유닛·계약**: Search(`discovery`)·DocModel/Summary(`summarization`)·Citation(`citation_graph`) = Tool 소비(기존 계약) · Grounding(`shared/ports`·페이즈 3 통합 Agent Validator) · CostGuard(`shared/ports`) · **연구아이디어 유닛 = 소비자**(본 질문지가 정하는 **근거 출력 DTO**가 §공유 계약 동결의 입력).
**근거 SSOT**: 재인셉션 차터 `inception/plans/reinception-2026-06-charter.md`(페이즈 4·§4·D2/D5) · 코드 베이스라인 `inception/reverse-engineering/code-baseline-2026-06.md`(§3 계약 인벤토리) · 포트 계약 초안 참조(구 `agent-tool-port-contract-draft.md` — 본 질문지로 4→5 번호·내용 확정).
**답변 상태**: 🟡 **초안 — 화랑님/팀 답변 대기.** 각 질문은 `[Answer]: (미정)` + **권장**을 병기. HOW(스키마 깊이·임계 수치)는 Functional/NFR Design으로 추가 이월 가능.

> ⚠️ **계약 선행 주의(D5)**: Q1~Q3(근거 산출물·근거표·**근거 출력 DTO**)은 연구아이디어 유닛(5)이 잠그는 **공유 계약의 심장**이다. 이 셋이 확정돼야 `shared/`에 evidence DTO를 동결하고 4·5 병렬이 풀린다(차터 §4.1).
