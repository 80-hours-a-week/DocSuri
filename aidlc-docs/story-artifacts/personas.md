<a id="p1"></a>
## P1. 박지훈 — AI 박사과정 전문가형

| 항목 | 값 |
|---|---|
| 나이/성별 | 28세 / 남 |
| 직무 | AI 대학원 박사과정 4년차, 졸업논문 준비 중 |
| 기술 친숙도 | 상급 (PyTorch · 임베딩 · 트랜스포머 등 핵심 어휘 능숙) |
| 주 사용 디바이스 | 데스크톱 (논문 PDF 관리), 보조 태블릿 |
| 한국어/영어 비중 | 영문 논문 정독 가능, 그러나 정리·문서화는 한국어 선호 |
| 가독성 수준 | 학술 전문 어휘 허용 |

### 사실 (Facts)
- 자신의 분야에서 하루 수백 건의 논문이 새로 공개된다.
- 일주일에 **10시간 이상**을 문헌조사에 쓴다 (전체 업무의 약 20%).
- 찾은 논문조차 코드·데이터 부재로 **재현이 어려운 경우가 흔하다**.

### 행동 (Behaviors)
- arXiv, Semantic Scholar, Google Scholar를 평행 검색.
- 생성형 AI에 "이 주제 이미 있어?" 같은 자연어 질문을 던지지만 출처 부족으로 신뢰가 낮다.
- 노션·옵시디언에 인용 메모를 축적.

### 목표 (Goals)
- 자신의 졸업논문 주제가 **이미 존재하는지(novelty)** 빠르게 확인.
- 분야의 **트렌드를 한눈에** 잡는다.
- 레퍼런스로 쓸 수 있는 **재현 가능한** 논문을 우선적으로 발견.

### 골칫거리 (Pains)
- DB 검색 결과의 신호/잡음비가 낮음.
- 논문 전체를 읽기 전에는 가치가 가늠이 안 됨.
- 검증에 시간이 오래 걸려 실제 연구 시간을 잠식.

### 성공 신호 (What "win" looks like)
- 본인 주제에 대한 *유사도 상위 5개 논문 + 차별성 노트*를 30분 안에 받는다.
- 인용 트리 1-hop 시각화로 **핵심 선행 연구**를 1분 안에 식별.

<a id="p2"></a>
## P2. 김민서 — AI 입문 학부생형

| 항목 | 값 |
|---|---|
| 나이/성별 | 21세 / 여 |
| 직무 | 4년제 대학 컴퓨터공학과 4학년, 졸업프로젝트로 AI 주제 탐색 |
| 기술 친숙도 | 중급 (Python·기초 ML 강의 수강, 트랜스포머·RAG 등 고급 개념은 학습 중) |
| 주 사용 디바이스 | 노트북 위주, 모바일 보조 |
| 한국어/영어 비중 | 영문 논문은 부담, **한국어 번역·요약 강한 선호** |
| 가독성 수준 | 학부 1~2학년 수준 한국어 (수식·전문 약어는 풀어쓰기 요구) |

### 사실 (Facts)
- AI 관련 정규 과목은 1~2개 수강했으며, 인터넷 강의로 보충 학습 중.
- 졸업프로젝트는 학부 수준의 토이 모델 구현 + 보고서 작성이 목표.
- 논문 한 편을 처음부터 끝까지 읽어 본 경험이 적다.

### 행동 (Behaviors)
- 생성형 AI에 "트랜스포머가 뭐예요?"처럼 **자연어 질문**을 적극적으로 사용.
- 영문 PDF는 우선 **번역기**부터 통과시킨다.
- 직접 검색은 막연해서 회피, 친구·튜터의 추천 논문을 따라 읽는다.

### 목표 (Goals)
- AI의 **본질·핵심 개념**을 정확히 잡는다.
- 자기 프로젝트에 **수준에 맞는 레퍼런스 논문 3~5편**을 확보.
- 어려운 수식·용어를 **이해 가능한 한국어**로 변환.

### 골칫거리 (Pains)
- 논문 문체와 수식이 진입 장벽.
- 어디서부터 시작해야 할지 모름 (탐색의 시작점 부재).
- 영어 약어와 인용 표기 해독에 시간 소요.

### 성공 신호 (What "win" looks like)
- 자기 프로젝트 주제를 한국어 자연어로 입력 → **학부 수준 적합 논문 3편**과 **각 논문 5문장 한국어 요약**을 받는다.
- 핵심 수식을 골랐을 때 한국어 풀이로 즉시 설명을 받는다.

## 두 페르소나의 시스템 시사점

| 차원 | 박지훈 | 김민서 |
|---|---|---|
| 입력 모달 | 영문 자연어/논문 제목 | 한국어 자연어 |
| 결과 깊이 | 상위 20개 + 메타데이터 풍부 | 상위 3~5개 + 짧은 한국어 요약 |
| 요약 톤 | 전문 어휘 유지 | 학부 수준 풀어쓰기 |
| 차별화 분석 | 본인 초안 vs 유사 5편 *novelty* 보고 | 가벼운 *"학부 수준에서 새로움"* 점검 |
| 인용 트리 | 1-hop 시각화 필수 | 영향력 큰 후속 논문 Top 3 추천형으로 단순화 |

> 같은 검색·요약 파이프라인을 공유하되, **출력 레이어**에서 페르소나 분기. 이 제약이 E2(Comprehend) Epic의 핵심 설계 결정으로 직결된다.
