# 05. 유사 논문 탐색 (Similar Paper Exploration)

> 사용자가 자기 연구 주제/관심 논문/현재 보고 있는 논문을 기준점으로 등록 → 시스템이 의미 유사 논문을 탐색·정렬·*"왜 유사한지"* 설명.

---

## 1. 핵심 요소

- **기준점(seed) 유형**: (a) 자유서술 연구 주제, (b) 특정 논문 1편, (c) 사용자 프로필(이전 검색·읽은 논문 누적), (d) 사용자의 미발표 초고.
- **임베딩 비교**: seed → 임베딩 → 벡터DB ANN 검색. 코사인 + 메타데이터 필터(연도, 분야, 인용수).
- **다축 정렬**:
  - 의미 유사도 (임베딩 cos)
  - 기여 유사도 (방법론/결과/문제 정의 중 어디가 닮았는지)
  - 시기 가중치 (최신 우선 / 클래식 우선)
  - 인용 거리 (인용 그래프 hop)
- **LLM 설명자**: "이 논문이 유사한 이유"를 2-3 문장 + anchor로 설명. 단순 점수가 아닌 "어디가" 유사한지.
- **반례(dissimilar) 모드**: "비슷해 보이지만 결정적으로 다른" 논문도 함께 제시 → 차이점 학습.

---

## 2. 주요 문제

- **Cold start**: 신규 사용자는 프로필 데이터가 없어 자유서술 seed에만 의존. 한 단락 자유서술의 임베딩 품질이 결정적.
- **임베딩의 한계**: 의미 유사도는 *문체*나 *분야 용어*에 끌릴 수 있음 — "강화학습 + 로봇팔" seed가 진짜 의미 유사한 "RL + manipulation" 대신 단순 같은 키워드 사용한 논문에 끌림.
- **다양성 vs 정확도**: top-K가 모두 같은 저자/그룹의 시리즈일 수 있음. MMR 같은 다양성 정규화 필요.
- **세부 분야 어휘**: 같은 개념을 다른 분야에서 다른 용어로 부름 (CV의 "attention", NLP의 "attention", 생물학의 "attention"). 분야 필터링 또는 cross-domain 매핑 필요.
- **"유사 이유" 설명의 신뢰성**: LLM이 그럴듯한 유사 이유를 만들어내는데 anchor가 약하면 검증 불가.

---

## 3. 파이프라인 설계 & 기술 스택

```
seed 입력 (텍스트 or paper_id)
    │
    ▼
┌─────────────────────────┐
│ Seed Embedder           │  텍스트 → 임베딩
│  - 자유서술: 직접 임베딩
│  - paper_id: 청크 mean + 메타데이터
│  - 프로필: 최근 N건 가중 평균
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ ANN Search              │  Qdrant top-K (K=100)
│  + 메타데이터 필터       │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ MMR Diversifier         │  λ-trade off diversity vs relevance
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Re-scorer (다축)        │  의미 + 기여 + 시기 + 인용거리
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Explainer (LLM)         │  각 결과에 "왜 유사한가" 2-3문장
│  + anchor 부여           │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Dissimilar Pair Finder  │  근접하지만 결정적 차이 있는 논문
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ UI                      │  카드 그리드 + 유사/차이 토글
└─────────────────────────┘
```

### 기술 스택

| 레이어 | 선택 | 비고 |
|---|---|---|
| 임베딩 | `text-embedding-3-large` 또는 SPECTER2(학술 특화) | SPECTER2가 학술 도메인 SOTA |
| ANN | Qdrant (HNSW) | 메타데이터 필터링 강력 |
| 다양성 | MMR (Maximal Marginal Relevance) λ=0.7 | |
| 기여 유사도 | 별도 임베딩 (논문의 contribution sentence만) | sentence extractor + 임베딩 |
| 인용 거리 | OpenAlex citation graph BFS | |
| Explainer | Claude Sonnet (정확도 우선) | seed + candidate 비교 |
| UI | React + react-flow (관계 시각화 옵션) | |

---

## 4. 차별화 포인트

- **"왜" 유사한지 설명 + anchor**: 대부분의 유사 논문 도구(Connected Papers, Litmaps)는 그래프와 점수만 제공. 본 시스템은 "이 논문이 seed의 §3 방법론과 닮았다, 결과 §5는 다르다"처럼 *구체적 위치*를 짚어줌.
- **반례(dissimilar) 모드**: 학습 자료로서 가치. "비슷해 보이지만 다른" 논문은 사용자가 자기 연구의 차별점을 정의할 때 결정적.
- **기여 유사도 분리**: 단일 임베딩 cos가 아니라 "문제 정의 / 방법 / 결과" 3축으로 분리. 사용자가 "방법은 닮았지만 결과가 완전히 다른" 논문을 빠르게 찾을 수 있음.

---

## 5. 위험 요소

- **임베딩 모델 의존성**: SPECTER2는 자체 호스팅 가능하지만 운영비. OpenAI/Voyage는 API 의존 (요금·정책 변동).
- **유사도 점수 신뢰성**: 사용자가 "왜 이게 top1인지" 납득 못하면 시스템 신뢰 무너짐. Explainer가 결정적이지만 그 자체가 환각 위험.
- **벡터DB 인덱스 크기**: 모든 등록된 논문을 ANN으로 검색하려면 인덱스가 클수록 비싸짐. shard·partition·tier 전략 필요.
- **저자/그룹 편향**: top-K가 한 그룹의 시리즈로 채워지면 사용자가 "탐색"이 아니라 "정찰" 모드로 빠짐. MMR + 저자 dedupe 보장.
- **언어 편향**: 영문 임베딩은 영어 논문에 강함. 한·중·일 논문은 임베딩 품질 낮음.

---

## 6. 예상 비용

### 단위 비용 (1 seed 탐색)

| 단계 | 비용 |
|---|---|
| Seed 임베딩 | $0.0001 |
| ANN 검색 | $0 (셀프호스트) |
| Re-score | LLM 미사용 — $0 |
| Explainer (top-10에 대해) | ~$0.05 (Sonnet, 청크 fetch 포함) |
| Dissimilar finder | ~$0.02 |
| **합계/탐색** | **~$0.07** |

### MVP (월 5,000 탐색)

- $0.07 × 5,000 = **$350/월**
- Vector DB 인덱스 (100만 청크): ~$60/월
- **총 ~$410/월**

### 스케일업 (월 50만 탐색)

- $0.07 × 500,000 = **$35,000/월**
- Vector DB (1억 청크, sharded): ~$1,500/월
- **총 ~$36,500/월**

> 비용 핵심은 Explainer 호출 수. top-10이 아닌 top-5만 설명 → 절반 절감. UI를 "보고 싶은 카드 클릭 시 설명 생성" lazy 모드로 가면 80% 절감 가능.
