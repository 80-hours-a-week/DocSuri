# 06. 연구 공백 분석 & 아이디어 제안 (Research Gap & Idea Generation)

> 사용자가 지정한 분야(또는 자유서술 주제)에 대해 시스템이 해당 분야의 corpus를 분석 → 미해결 문제/연구 공백/혼재된 결과를 식별 → 새로운 연구 아이디어 후보를 제안.
> **원본 #6 분할 a**. (자기 프로젝트 통합 분석은 07로 분리.)

---

## 1. 핵심 요소

- **분야 정의**: 사용자가 키워드 + 시기 + 출판처 + 제외 키워드를 조합해 분야 범위(scope)를 정의.
- **Corpus 수집**: 정의된 scope의 논문 N편(예: 500-2000편)을 검색·인입.
- **토픽 모델링**: 임베딩 클러스터링으로 sub-topic 자동 식별.
- **공백 신호**:
  - **빈도 vs 인용 gap**: 자주 다뤄지지만 인용 적은 영역 = 합의 안 됨.
  - **claim 충돌**: 같은 질문에 다른 답을 하는 논문군.
  - **저빈도 sub-topic**: 인용은 많지만 후속 연구가 적은 영역.
  - **방법 누락**: 일반적인 기법이 적용되지 않은 sub-topic (e.g. 어떤 분야에 transformer 미적용).
  - **데이터·벤치 공백**: 평가 benchmark가 부재한 영역.
  - **시계열 단절**: 활발하다가 갑자기 활동 사라진 영역.
- **아이디어 생성기**: 식별된 공백 × 사용자 제약(시간, 자원, 도메인) → 아이디어 N개 + feasibility 점수.
- **anchor 강제**: 모든 공백 주장은 corpus 내 구체 논문 anchor로 뒷받침.

---

## 2. 주요 문제

- **"공백"의 모호함**: 진짜 미해결 vs 단순히 검색에 안 잡힌 것 vs 비공개 산업체 R&D 진행 중 — LLM이 구별 어려움.
- **환각 위험 최상위**: "아무도 안 했다"라는 주장은 검증이 본질적으로 negative — 시스템이 못 찾은 거지 진짜 없는 게 아닐 수 있음.
- **분야 정의 sensitivity**: 키워드 한 단어 차이가 corpus를 크게 흔듦 → 공백도 달라짐.
- **아이디어의 실현성 평가**: LLM이 "트랜스포머를 단백질 접힘에 적용해보세요" 같은 이미-된 제안 또는 무의미한 조합 생성.
- **연구 윤리**: 익명 사용자의 미공개 아이디어를 시스템이 다른 사용자에 제안하면 도용. 사용자 입력 격리 필수.
- **분야 전문성 부족**: LLM의 학술 도메인 지식은 일반적. niche 분야의 *암묵적* 합의를 모를 수 있음.

## 3. 파이프라인 설계 & 기술 스택

```
사용자 분야 정의 (scope)
    │
    ▼
┌─────────────────────────┐
│ Corpus Collector        │  → 01 검색 파이프라인 (대용량 모드)
│  - N=1000편 수집         │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Chunk + Embed           │  논문 → 청크 → 임베딩
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Topic Modeler           │  HDBSCAN / BERTopic / LLM 라벨링
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Signal Extractors       │
│ - claim conflict        │  → 같은 질문 다른 답 탐지
│ - method gap             │  → method × topic 매트릭스
│ - benchmark gap          │  → 토픽별 benchmark 언급 부재
│ - temporal gap           │  → 활동 시계열 단절
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Gap Synthesizer (LLM)   │  signals → 자연어 공백 진술 + anchor
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Idea Generator (LLM)    │  gap × 사용자 제약 → 아이디어 후보 N개
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Feasibility Scorer      │  자원/시기/난이도 추정
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Verifier                │  공백 주장 ↔ corpus evidence
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ UI: 공백 맵 + 아이디어  │  topic 클러스터 시각화
└─────────────────────────┘
```

### 기술 스택

| 레이어 | 선택 | 비고 |
|---|---|---|
| Corpus 수집 | 01 재사용 + bulk 모드 | 1000편 = ~30분 처리 |
| 토픽 모델링 | BERTopic (UMAP+HDBSCAN) | 자동 클러스터·라벨링 |
| Signal 추출 | sklearn + 자체 휴리스틱 + LLM | 신호별 다른 도구 |
| Claim conflict | embedding 기반 stance detection | 자체 분류기 fine-tune 옵션 |
| Gap Synthesizer | Claude Opus 4.7 | 추론 깊이가 필요한 단계 |
| Idea Generator | Claude Opus + 다중 sampling | n=10 후보 생성 후 dedupe |
| Feasibility | LLM + rule (자원/시기 추정 룰) | |
| 시각화 | Plotly / UMAP 2D projection | |

---

## 4. 차별화 포인트

- **신호 다축화**: 단순 LLM "아이디어 내줘"가 아니라, claim conflict / method gap / benchmark gap 같은 **구체적·정량적 신호**가 아이디어 생성의 입력. 환각 감소.
- **anchor 강제 공백 진술**: "X 분야에서 transformer 미적용"이라는 주장은 *실제로 적용 안 한 N편*의 anchor를 첨부.
- **Feasibility scoring**: 아이디어에 "이건 박사논문 1년/3년/5년" 추정. 단순 생성형 도구와 차별.
- **분야 정의 sensitivity 노출**: scope을 살짝 바꿔 다시 돌렸을 때 공백이 어떻게 달라지는지 보여줌 → 사용자가 자기 정의의 신뢰도를 가늠.

---

## 5. 위험 요소

- **환각의 가시성 부족**: "아무도 안 한 X"가 가장 매력적이지만 가장 위험. 모든 negative claim에 *적극적* 부정 검증 필요 (검색 보강 후 다시 검토).
- **사용자 입력 격리**: 같은 시스템 안에서 user A의 미공개 아이디어가 user B에게 노출될 가능성 (LLM 컨텍스트 공유). 세션·테넌트 격리 + LLM 메모리 사용 금지.
- **IP 분쟁**: 시스템이 제안한 아이디어를 사용자가 출판했을 때 시스템 다른 사용자에게도 같은 아이디어 제안된 경우 우선권 분쟁 가능. ToS 명시 필요.
- **분야 sensitivity**: 정의가 부정확하면 사용자는 잘못된 공백을 좇게 됨. 시스템이 "당신의 정의로는 N편이 잡혔고, 비슷한 정의 N+50편으로는 다음과 같이 달라집니다"를 제공.
- **운영 비용 폭증**: corpus 수집·임베딩·토픽모델링이 무겁다. 같은 분야 정의 캐시 강제.
- **niche 분야 무지**: LLM이 잘 모르는 도메인에서는 "공백"이라 부른 것이 사실 잘 알려진 부분. 도메인 전문가 검토 권장.

## 6. 예상 비용

### 단위 비용 (분야 1회 분석, 1000편)

| 단계 | 비용 |
|---|---|
| Corpus 수집 (검색+인입) | ~$5 (01의 PDF 파싱·임베딩 포함) |
| 토픽 모델링 | 컴퓨트 ~$0.5 |
| Signal 추출 | $0.3 |
| Gap Synthesizer (Opus) | ~$2 |
| Idea Generator (Opus × 10 sample) | ~$3 |
| Feasibility | ~$0.5 |
| Verifier | ~$0.7 |
| **합계/분석** | **~$12** |

### MVP (월 500 분석)

- $12 × 500 = **$6,000/월**
- 같은 분야 캐시 30% 가정 → 실제 **~$4,200/월**

### 스케일업 (월 1만 분석)

- $12 × 10,000 = **$120,000/월**
- 캐시 50% 가정 → **~$60,000/월**

> 이 기능은 11개 중 단일 호출 비용이 가장 비싼 축. **scope 캐싱**과 **incremental 분석**(이전 corpus + 신규 N편만 추가) 설계가 ROI를 결정. 단발성 SaaS 가격 ≥ $10/회.
