# 02. 논문 요약 (Summarization)

> 벡터DB에 인입된 논문 전문에서, 사용자가 요청한 길이/관점으로 **anchor가 부착된 한국어 요약**을 생성한다.
> 모든 문장은 원문 span을 인용한다 (`[§4.2 ¶3]`).

---

## 1. 핵심 요소

- **길이 프리셋 3종**: 1줄(TL;DR) / 한 문단(150-200자) / 한 페이지(800-1200자). 동일 논문에 재호출 시 길이만 바꿀 수 있음.
- **관점 프리셋**: "기여 중심" / "방법 중심" / "결과·실험 중심" / "비판적 검토" 4종.
- **anchor 강제**: 산출물의 모든 sentence에 `[§n.m]` 또는 `[p.X ¶Y]` 형식 인용이 의무 (`AGENTS.md §4.3` 정책 준수).
- **Prompt caching**: 논문 전문을 `cache_control: ephemeral`로 cached system 블록에 배치 → 같은 논문에 대한 두 번째 호출부터 입력 토큰 비용 ~90% 절감.
- **Verifier 횡단 호출**: 각 요약 문장이 생성될 때마다 anchor의 원문 span을 다시 가져와 LLM에 entailment 판정(4-way: SUPPORTED / PARTIALLY_SUPPORTED / UNSUPPORTED / NOT_FOUND)을 강제.
- **glossary 누적**: 번역과 공유하는 세션별 도메인 용어 사전 — 동일 용어는 요약 전반에 동일 한국어로 일관 표기.

---

## 2. 주요 문제

- **환각**: 요약은 "원문에 없는 것을 부드럽게 덧붙이는" 유혹이 강한 작업. anchor 의무화는 부분 해결이지만, anchor 자체가 잘못 부여되면 검증을 통과해버림.
- **긴 컨텍스트 한계**: 100-page 박사 학위 논문 / 80-page 서베이는 단일 컨텍스트에 들어가지 않을 수 있음. Claude 200k 컨텍스트로도 안전 마진을 두면 ~150k 토큰까지가 실용 한계.
- **표·수식·그림 캡션**: 텍스트 임베딩만으로 충분한 의미를 못 살리는 경우 (벤치마크 표가 핵심인 SOTA 논문).
- **섹션 불균형 요약**: Method 섹션을 과도하게, Discussion을 무시하는 경향 → 관점 프리셋으로 부분 완화.
- **재요약 일관성**: 같은 논문 같은 프리셋을 두 번 호출했을 때 결과가 크게 다르면 사용자 신뢰가 떨어짐. seed 고정 + temperature 낮춤.

---

## 3. 파이프라인 설계 & 기술 스택

```
[req] paper_id + length_preset + angle_preset
    │
    ▼
┌─────────────────────────┐
│ Retriever               │  논문 전문(구조화 md) + 관련 청크 fetch
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Prompt Builder          │
│  [CACHED] system: §6    │  ← AGENTS.md 행동 규약
│  [CACHED] system: 전문  │  ← cache_control: ephemeral
│  [FRESH ] user: 요청    │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Claude Sonnet/Opus      │  요약 + anchor 부여
└─────────┬───────────────┘
          │ summary sentences with anchors
          ▼
┌─────────────────────────┐
│ Anchor Validator        │  각 anchor가 실제 span을 가리키는지
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Verifier (횡단 관심사)  │  sentence × evidence → 4-way label
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Glossary Updater        │  새 한국어(영문) 매핑 누적
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ UI Renderer             │  요약 + anchor hover + verify badge
└─────────────────────────┘
```

### 기술 스택

| 레이어 | 선택 | 비고 |
|---|---|---|
| 요약 LLM | Claude Sonnet 4.6 (기본) / Opus 4.7 (긴 논문/critical 관점) | prompt cache 활용 |
| Verifier LLM | Claude Haiku 4.5 | entailment 판정은 저렴한 모델로 충분 |
| 컨텍스트 관리 | Anthropic SDK `cache_control` | TTL 5분 기본, beta extended cache 옵션 |
| 임베딩 검색 | Qdrant ANN | 청크 fetch 보조 (긴 논문에서 일부만 컨텍스트화 필요 시) |
| 길이 제어 | system prompt + post-trimmer | 토큰 카운트 측정 후 컷 |
| 일관성 | temperature 0.2, top_p 0.9, seed 고정 | |
| 출력 포맷 | structured JSON (sentence[], anchors[], verify_label[]) | UI 렌더링용 |

---

## 4. 차별화 포인트

- **요약 ↔ anchor 양방향 인터랙션**: 다른 요약 도구는 텍스트만 반환하지만, 본 시스템은 각 문장 hover 시 원문 span 하이라이트 + verify 라벨 표시. "어디서 나온 주장인가" 추적이 한 클릭.
- **관점 프리셋**: GPT 기반 도구 다수가 "한 가지 요약"만 제공. "비판적 검토" 모드(가정·한계·재현 가능성 의문점)는 학술 사용자에게 차별적 가치.
- **세션 glossary 누적**: 시리즈 논문(예: BERT → RoBERTa → ALBERT) 연속 요약 시 용어 표기 통일이 자동 유지.

---

## 5. 위험 요소

- **anchor 위조**: LLM이 "그럴듯한 §n.m"을 만들어내는 경우. Anchor Validator가 실제 존재하는 anchor와 매칭하지 않으면 거부 + 재요청.
- **Prompt cache TTL**: 5분 TTL 기본이라 사용자가 한 논문을 20분 동안 여러 번 다른 관점으로 보면 캐시 미스. extended cache(beta, 1h)로 완화 가능하나 비용 증가.
- **출판사 ToS**: 일부 출판사는 자동 요약 서비스 제공을 ToS에서 제한. 사용자 본인이 정당하게 접근한 논문에 한해 요약을 수행한다는 정책 필요.
- **요약 vs 표절 경계**: 한국어 요약이 원문 영어 문장의 직역에 가까우면 "번역+표시"인지 "요약"인지 모호. 100자 이상 연속 직역 금지 룰 추가 필요.
- **비판적 검토 모드의 신뢰성**: LLM이 "한계점"을 만들어낼 때 anchor가 약함 (원문에 명시되지 않은 추론). 별도 verify 모드 필요.

## 6. 예상 비용

### 단위 비용 (논문 1편 요약, 50p 평균)

| 단계 | 토큰 | 비용 |
|---|---|---|
| 입력(논문 전문, **첫 호출**) | ~80,000 | $0.24 (Sonnet 4.6 캐시 쓰기 단가 적용) |
| 입력(논문 전문, **2회차 이후 캐시 히트**) | ~80,000 | $0.024 (90% 절감) |
| 입력(질의 + 시스템) | ~2,000 | $0.006 |
| 출력(요약) | ~500-1500 | $0.0075~$0.0225 |
| Verifier (Haiku, sentence별) | ~10 sentences × 1k tokens | ~$0.01 |
| **첫 요약** | | **~$0.28** |
| **재요약 (캐시 히트)** | | **~$0.05** |

### MVP (월 1만 요약)

- 70%가 캐시 히트 가정 → 평균 단가 ~$0.12 × 10,000 = **$1,200/월**
- Verifier 별도 ~$100/월

### 스케일업 (월 100만 요약)

- 평균 단가 동일 가정 → **$12,000/월**
- Volume discount 협상 가능 영역

> 비용 최적화 키: **prompt caching 유지율**. 사용자가 같은 논문을 5분 안에 재호출하는 행동 패턴이 비용을 결정.
