# 03. 논문 번역 (Translation)

> 벡터DB에 저장된 논문에서 사용자가 선택한 span(문장/문단/섹션)을 한국어로 학술체 번역. 도메인 용어는 `한국어(English)` 병기.

---

## 1. 핵심 요소

- **Span 단위 번역**: 사용자가 영문 span 선택 → 해당 단위만 번역. 논문 전체 일괄 번역은 비용·품질 모두 비효율 (필요 시 점진적 누적).
- **학술체 톤 강제**: 의역 최소화, 종결형 통일(`-한다`체), 능동/수동 원문 보존, 인용 형식(`Smith et al., 2023`) 보존.
- **도메인 용어 사전(glossary)**: 세션별 누적. 한 번 결정된 한국어 표기는 동일 세션 내 일관 유지 (예: `attention → 주의(attention)` 결정 시 이후 모든 등장에 동일 적용).
- **첫 등장 글로싱**: 도메인 용어의 *세션 내 첫 등장* 시 `한국어(English)` 병기, 이후 등장은 한국어만 (또는 사용자 옵션).
- **수식·기호·표·인용 보존**: LaTeX 수식과 `[12]` 인용 번호, 표 구조는 통과(pass-through), 번역하지 않음.
- **anchor 유지**: 번역 결과의 각 문장이 원문 span의 정확한 위치를 유지하도록 1:1 매핑 보존 (UI에서 토글 시 영-한 대조 보기 가능).

---

## 2. 주요 문제

- **용어 일관성 vs 자연스러움**: 동일 영어 단어가 문맥에 따라 다르게 번역돼야 자연스러운 경우가 있음 (`model` = 모델 / 모형 / 모방). glossary가 너무 엄격하면 어색, 너무 느슨하면 일관성 깨짐.
- **긴 문장 구문 재배열**: 영어의 관계절·동격 구문을 한국어로 풀어쓰면 어순이 크게 바뀜 → anchor 1:1 매핑 깨짐 위험.
- **수식 인라인**: `we maximize $\log p_\theta(x)$ over` 같은 인라인 수식의 번역 경계 처리.
- **저자 고유 표기**: 새로운 용어를 저자가 만든 경우 (예: "FlashAttention"), 번역할지 음역할지 그대로 둘지.
- **표·캡션의 미니멀 텍스트**: 표 헤더 한두 단어는 문맥 정보가 부족해 오역 위험.
- **수동 vs 능동의 한국어 자연성**: 영어 학술체는 수동태가 많음. 한국어 학술체는 능동/수동 혼용이 자연. 원문 보존 vs 가독성 trade-off.

---

## 3. 파이프라인 설계 & 기술 스택

```
[req] paper_id + span (section_id, char_start, char_end) + options
    │
    ▼
┌─────────────────────────┐
│ Span Resolver           │  span → 원문 텍스트 + 주변 컨텍스트 ±200자
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Glossary Lookup         │  세션 누적 사전에서 매칭 용어 추출
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Prompt Builder          │
│  [CACHED] system §6     │  ← 행동 규약
│  [CACHED] system 전문   │  ← 논문 컨텍스트 (다른 단계 공유)
│  [CACHED] system gloss  │  ← 세션 glossary
│  [FRESH ] user span     │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Claude Sonnet           │  학술체 번역 + 첫등장 글로싱
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Post-processor          │
│ - LaTeX 수식 패스스루   │
│ - 인용 번호 보존        │
│ - 종결형 통일           │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Glossary Updater        │  새로 결정된 용어 매핑을 사전에 추가
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Verifier                │  번역 ↔ 원문 의미 보존 확인
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ UI: side-by-side        │  영-한 1:1 대조 보기
└─────────────────────────┘
```

### 기술 스택

| 레이어 | 선택 | 비고 |
|---|---|---|
| 번역 LLM | Claude Sonnet 4.6 | 학술체 한국어 품질 최상위 |
| 수식 보호 | regex 마스킹 → 번역 후 복원 | `$...$`, `\\[...\\]` 패턴 |
| Glossary 저장 | Redis hash (session_id별) | TTL 24h |
| 후처리 | python 종결형 변환기 (선택) | `-합니다`체 → `-한다`체 |
| Verifier | Claude Haiku | 의미 보존 entailment |
| UI 다이프 | `diff-match-patch` | 영-한 정렬 시각화 |

---

## 4. 차별화 포인트

- **세션 glossary 누적**: 동일 사용자가 같은 분야 5편을 연달아 번역해도 `transformer → 트랜스포머`, `attention → 주의(attention)` 통일. 일회용 번역기와 결정적 차이.
- **anchor 양방향 토글**: 한국어 문장 hover → 원문 영어 span 강조 / 영어 hover → 한국어 강조. 독해 학습 도구로도 활용 가능.
- **수식·인용 보존 정확도**: GROBID로 미리 분리된 수식 노드를 번역 대상에서 제외 → 수식이 깨지지 않음. 단순 LLM 직번역과 차별.
- **`AGENTS.md §6.2 glossing rule` 준수**: 첫 등장 글로싱 규칙이 시스템 헌법 수준으로 강제 → 사용자가 원문과 한국어를 모두 기억하기 쉬움.

---

## 5. 위험 요소

- **장문 번역의 anchor 흐트러짐**: 문단 단위 번역 시 어순 재배치로 1:1 sentence 매핑이 깨지면 anchor가 무의미. 최대 3문장 청크로 제한 권장.
- **glossary 오결정의 전파**: 첫 번역에서 잘못 결정된 한국어 표기가 같은 세션 내 50번 반복되면 신뢰 회복 비용 큼. 사용자 수정 시 즉시 glossary 갱신 + 과거 번역 재처리 옵션.
- **수식 마스킹 실패**: 인라인 수식이 백쿼트로 둘러싸인 경우, 코드블록으로 시작·끝나지 않는 경우 등 엣지 케이스에 정규식 fragility.
- **LLM의 의역 경향**: temperature 낮춰도 학술적 정확성보다 자연스러움을 택하는 경우 있음. system prompt에 "literal-first, idiomatic-second" 명시 필요.
- **Verifier 비용 누적**: 모든 번역 결과에 횡단 verify를 호출하면 비용이 번역의 30-50% 추가됨. sampling verify(10% 무작위) 옵션 검토.

---

## 6. 예상 비용

### 단위 비용 (1문단 ~150 단어 번역)

| 단계 | 토큰 | 비용 |
|---|---|---|
| 입력(전문 캐시 히트) | ~80,000 | $0.024 |
| 입력(질의+glossary) | ~3,000 | $0.009 |
| 출력(한국어 번역) | ~300 | $0.0045 |
| Verifier (Haiku) | ~2,000 | $0.001 |
| **합계/문단** | | **~$0.04** |

> 캐시 미스(첫 호출)는 ~$0.27. 같은 논문에서 N문단을 연속 번역 시 첫 ~$0.27 + (N-1) × $0.04.

### MVP (월 5만 문단 번역)

- 캐시 히트율 80% 가정 → 평균 $0.05 × 50,000 = **$2,500/월**

### 스케일업 (월 100만 문단)

- 평균 $0.05 × 1,000,000 = **$50,000/월**
- 대량 사용 시 Volume discount 또는 Haiku 다운그레이드 옵션 검토 (품질 trade-off)

> 캐시 히트율과 verify sampling 비율이 비용의 결정 변수. 사용자가 한 논문을 5분 안에 여러 부분 번역할수록 단가 저하.
